# ADR-005: Feature-Sliced Design для структуры фронтенда

## Статус
Принято

## Контекст
Нужна структура папок для React-фронтенда, которая масштабируется и не превращается в хаос при росте проекта.

## Решение
Feature-Sliced Design (FSD) — методология с жёсткой иерархией слоёв:

```
src/renderer/src/
  app/       # точка входа, роутер, глобальные провайдеры
  pages/     # компоненты страниц
  widgets/   # составные блоки (navbar, create-recipe форма)
  features/  # переиспользуемые фичи со своим состоянием (amount)
  entities/  # бизнес-сущности и их сторы (recipe)
  shared/    # UI-компоненты, helpers, типы, конфиг
```

Правило: слой может импортировать только из слоёв **ниже** себя.

## Альтернативы
- **Flat / by-type** (`components/`, `utils/`, `hooks/`) — быстро становится неуправляемым
- **Atomic Design** — хорошо для UI-кит, плохо для бизнес-логики
- **Ducks** — подходит для Redux-проектов

## Последствия
**Плюсы:** понятное место для каждого файла; предотвращает циклические зависимости; хорошо масштабируется

**Минусы:** требует дисциплины и понимания методологии; overhead для маленьких проектов

**Когда пересмотреть:** FSD подходит и для веб-версии проекта, пересматривать не нужно
