# ADR-006: Централизованные константы IPC-каналов

## Статус
Принято

## Контекст
IPC-каналы — это строки вроде `'recipes:getAll'`, которые используются и в main (контроллер), и в preload. Если писать их в двух местах, легко допустить опечатку или рассинхронизацию.

## Решение
Единый файл `src/main/ipc-channels.ts` со всеми константами:

```ts
export const RECIPES = {
  GET_ALL: 'recipes:getAll',
  ADD: 'recipes:add',
  REMOVE: 'recipes:remove',
};
```

Импортируется и в контроллер, и в preload. Соглашение по именованию: `entity:action`.

## Альтернативы
- Строки напрямую в коде — быстро, но хрупко
- Enum — менее удобен для группировки по сущности

## Последствия
**Плюсы:** единое место для всех каналов; переименование в одном месте; служит документацией API между процессами

**Минусы:** файл импортируется в preload, значит попадает в preload-бандл (незначительно)

**Когда пересмотреть:** не нужно — простое и надёжное решение
