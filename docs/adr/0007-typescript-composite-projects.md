# ADR-007: Раздельные tsconfig для main и renderer

## Статус
Принято

## Контекст
В проекте есть два принципиально разных окружения: Node.js (main, preload) и браузер (renderer). У них разные доступные API, модульные системы и алиасы путей.

## Решение
TypeScript Project References — три конфига:

- `tsconfig.json` — корень, только ссылки на два дочерних
- `tsconfig.node.json` — покрывает `src/main/`, `src/preload/`, `src/shared/`; алиасы `@-electron/*` и `@shared/*`
- `tsconfig.web.json` — покрывает `src/renderer/`, `src/shared/`; алиасы `@/*` и `@shared/*`

`src/shared/` включён в оба конфига — это единственное место, откуда типы шарятся между main и renderer.

## Альтернативы
- Один общий tsconfig — renderer мог бы импортировать Node.js модули без ошибок компиляции
- Монорепо с отдельными пакетами — избыточно для одного проекта

## Последствия
**Плюсы:** TypeScript не даст импортировать `fs` или `electron` в renderer; разные алиасы исключают путаницу; инкрементальная компиляция

**Минусы:** три файла вместо одного; нужно помнить, в какой конфиг добавлять новые папки

**Когда пересмотреть:** не нужно — это стандартный паттерн для electron-vite
