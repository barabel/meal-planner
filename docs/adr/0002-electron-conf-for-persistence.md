# ADR-002: electron-conf для персистентности

## Статус
Принято

## Контекст
Нужно хранить данные между запусками приложения. Это desktop-приложение с небольшим объёмом данных (рецепты пользователя).

## Решение
Использовать `electron-conf` — типизированное key-value хранилище на основе JSON-файла на диске.

```ts
const store = new Conf<StoreSchema>({ ... });
store.get('recipes'); // IRecipe[]
store.set('recipes', [...recipes, newRecipe]);
```

ID генерируется в main process через `crypto.randomUUID()`.

## Альтернативы
- **SQLite / better-sqlite3** — мощнее, но требует миграций и больше настройки
- **electron-store** — аналог, но `electron-conf` имеет встроенную TypeScript-поддержку
- **Прямой fs + JSON.parse** — низкоуровнево, нет типизации

## Последствия
**Плюсы:** простота, нет миграций, типизация из коробки, хорошо для MVP

**Минусы:** не подходит для больших объёмов данных; не поддерживает сложные запросы

**Когда пересмотреть:** при росте данных или необходимости поиска/фильтрации — мигрировать на SQLite или внешний API (что и является долгосрочным планом проекта)
