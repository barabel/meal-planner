# ADR-004: Zustand для управления состоянием

## Статус
Принято

## Контекст
Фронтенду нужно хранить список рецептов и синхронизировать его с main process через IPC. Нужен способ шарить это состояние между компонентами.

## Решение
Использовать Zustand — минималистичную стейт-библиотеку. Стор в `entities/recipe`:

```ts
const useRecipesStore = create((set, get) => ({
  recipes: [],
  fetchAll: async () => { ... },
  addRecipe: async (dto) => { ... },
  removeRecipe: async (id) => { ... },
}));
```

## Альтернативы
- **Redux Toolkit** — более verbose, overkill для одного стора
- **React Context + useReducer** — нет DevTools, сложнее async
- **TanStack Query** — хорошо для server-state, избыточно для IPC без кэширования

## Последствия
**Плюсы:** минимальный boilerplate; легко добавлять новые сторы; хорошая TypeScript-поддержка

**Минусы:** при большом количестве сторов нужна дисциплина, чтобы не превратить их в "глобальные переменные"

**Когда пересмотреть:** при миграции на веб-API — рассмотреть TanStack Query для кэширования и ревалидации
